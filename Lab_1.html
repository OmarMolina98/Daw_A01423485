Investigación

URL

URL en inglés significa Uniform Resource Locator por lo que en español se traduciría como Localizador Uniforme de Recursos, este recurso fue creado por Tim Berners-Lee y se uso por primera vez en 1991. En pocas palabras el URL es una dirección especial que se da a cada medio o recurso que se encuentra disponible dentro de la red para encontrar lo que queremos dentro del web. Para que pueda ser usado efectivamente se necesitan varios componentes los cuales son: https://, www, (nombre del dominio) y .com .



MIME

Los MIME Types (Multipurpose Internet Mail Extensions) son la manera standard de mandar contenido a través de la red. Los tipos MIME especifican tipos de datos, como por ejemplo texto, imagen, audio, etc. que los archivos contienen. Recuerde que debe utilizar el sufijo correcto para este tipo de archivo, dicha significado fue propuesto por la página “Soluciones VIA Internet” en el año 2019 [1].



Codigos de errores de http

En http existen diferentes códigos de estados los cuales sirven para que un servidor pueda retornar una cierta petición que el usuario acciono. Hoy en día la Internet Assigned Numbers Authority (IANA) tiene un registro de todos los códigos dentro de http. Existen 5 tipos de respuestas las cuales son:

*Respuestas informativas Este tipo de código da como significado que el servidor recibió ciertos encabezados y que el cliente tendrá que enviar el cuerpo de esta, salvo para pruebas.

• 100 Continue. El servidor ha recibido los headers del request y el cliente debería enviar el cuerpo de la respuesta.

• 101 Switching Protocols. El requester ha solicitado al servidor conmutar protocolos.



*Peticiones correctas

En este tipo de código hace que la petición sea recibida, entendida y aceptada de forma correcta.

• 200 OK. El request es correcto. Esta es la respuesta estándar para respuestas correctas.

• 201 Created. El request se ha completado y se ha creado un nuevo recurso.



*Redirecciones

En este tipo de código se le indica una cierta acción la cual necesita efectuarse por un user-agent para realizar la acción o petición. La acción se puede llevar a cabo por el user-agent sin una conexión directa con el usuario si y solo si el método que se proporciono en la segunda petición es por parte del GET o HEAD.

• 300 Multiple Choices. Es una lista de enlaces. El usuario puede seleccionar un enlace e ir a esa dirección. Hay un máximo de cinco direcciones.

• 301 Moved Permanently. La página solicitada se ha movido permanentemente a una nueva URI.



*Errores del servidor

Es para ciertos casos los cuales se piensa que el cliente erro alguna petición, a excepción de la petición HEAD.

• 500 Internal Server Error. Error genérico, cuando se ha dado una condición no esperada y no se puede concretar el mensaje

• 501 Not Implemented. El servidor o no reconoce el método del request o carece de la capacidad para completarlo. Normalmente es algo que se ofrecerá en el futuro, como un nuevo servicio de una API.

Todos los tipos de respuestas fueron propuestos en el año 2018 por Diego Lázaro.[2]

Para ver todos los tipos de respuestas visita su página.

https://diego.com.es/codigos-de-estado-http

*HTML5

Para empezar HTML significa “HyperText Markup Language, HTML5 provee básicamente tres características: estructura, estilo y funcionalidad. Nunca fue declarado oficialmente, pero, incluso cuando algunas APIs (Interface de Programación de Aplicaciones) y la especificación de CSS3 por completo no son parte del mismo, HTML5 es considerado el producto de la combinación de HTML, CSS y Javascript. Estas tecnologías son altamente dependientes y actúan como una sola unidad organizada bajo la especificación de HTML5, dicha propuesta fue propuesta por J.D. Gauchat en el año 2012 [3]



*PHP

PHP (acrónimo recursivo de PHP: Hypertext Preprocessor) es un lenguaje de código abierto sumamente popular el cual esta diseñado para el desarrollo web y que puede ser introducido en HTML. Este tipo de lenguaje se ejecuta en el servidor web, y antes de que se envíe la página a través de Internet al cliente.



*JSP

JSP es un acrónimo de Java Server Pages. Esta es una tecnología orientada a crear páginas web con programación en Java. Gracias a JSP podemos crear aplicaciones web que se pueda ejecutar en variados servidores web, de múltiples plataformas, ya que Java es por defecto un lenguaje multiplataforma. Las páginas JSP están compuestas de código HTML/XML mezclado con etiquetas especiales para programar scripts de servidor en sintaxis Java.



*AJAX

AJAX significa JavaScript asíncrono y XML (Asynchronous JavaScript and XML). Es un conjunto de técnicas de desarrollo web que permiten que las aplicaciones web funcionen de forma asíncrona, de igual forma Ajax se pueden crear páginas interactivas.

La diferencia entre Internet y World Wide Web es básicamente es que el Internet es el medio y la WWW es la web la cual hace posible que pueda correr el internet

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

El propósito de http es que transfiera los datos a través de Internet, GET es cuando queremos pedir un recurso a un servidor, HEAD tiene como propósito obtener info. de un cierto recurso sin regresar nada el registro, POST hace que la info. se envíe a través del body de un http request, PATCH se utiliza para aplicar cambios o modificaciones parciales de un recurso y por último DELETE como el mismo nombre lo dice sirve para borrar un recurso en específico.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

El método que se debe utilizar es el de POST porque no se están enviando parámetros

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

El parámetro que se utiliza es el GET.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
